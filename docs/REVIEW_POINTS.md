## 第2部　レビュー時のチェック観点

FBC Press 共通の `REVIEW_CHECKLIST.md`（`/Users/machida/dev/FBC_Press`）を土台に、本書固有の観点を足したもの。**好みでなく基準で見る。** 順番は重要度順。

### 1. 密度・固有性（最優先）
- [ ] 章の主張を「具体例・対比・根拠」の3点で支えているか。言い換えで段落を埋めていないか
- [ ] コード例・事実・人物が、前を踏まえて**積み上がって**いるか（並列に置いただけでないか）
- [ ] 読者が「だからこうなのか」と一段深く理解する瞬間が章内にあるか
- [ ] タイトルの語を変えても通る一般論になっていないか（その章にしか書けない中身か）

### 2. 本書の背骨（不自由→自由）
- [ ] 一つの不自由を解き、一つの自由を勝ち取る形になっているか（不自由の地図と対応）
- [ ] **「自由」を本文で乱発していないか**（第6章・終章以外は着地文のみ）
- [ ] 章末の着地文が、勝ち取った自由を一文で言い切っているか／安っぽくないか
- [ ] 不自由（問題側）は本文で名指し、自由（勝ち取り側）は着地に温存できているか

### 3. 6拍子と崩し
- [ ] 6拍（問題→最初の解決策→失敗→新しい考え→今の当たり前→まだ終わらない）が立っているか
- [ ] 失敗が各章に最低一つあり、人物を「いい人」で終わらせていないか
- [ ] ⑤に象徴場面と誤解が各一つ。誤解は独立コーナーでなく⑤の内側で解いているか
- [ ] ⑥が「閉じた問い／開いた問い」のどちらかを示して締めているか
- [ ] 崩し（A〜E）が事前割り当てでなく章のエネルギーから来ているか／**隣接章で重複していないか**

### 4. 文体・体裁
- [ ] 見出しが です・ます調になっていない（体言止め・疑問形・である調）
- [ ] **和欧間スペース**（`HTML は`／`1996 年`）。全角約物とは空けない
- [ ] **章末の定型句が全章同じ形になっていないか**（※ 2026-06-30 に「最初の問いに戻ろう」「向きだけは揺るがない」を全章ばらした。今後の加筆でも再発させない）
- [ ] 「その話は、次の章で。」は**意図的な決め台詞**として残す（第1・4章サンプル準拠。これは反復OK）
- [ ] AIっぽい言い回し（効いてくる／正面から／という見方／地味／解像度／腑に落ちる）が出ていないか
- [ ] `## まとめ` 節を作っていない。見出しなしの結び＋次章橋になっているか
- [ ] 章の導入が全章同型になっていないか（場面・問い・歴史でばらけているか）

### 5. 事実・出典
- [ ] 固有名・年号・逸話の精度（`docs/notes/fact-check.md`）。`○` 項目は刊行前に一次資料へ
- [ ] 旧説・旧バージョンを誤差なく条件限定しているか（「絶対ダメ」でなく「いつ痛むか」）
- [ ] 本文に固有名を出した箇所は、巻末注／年表と整合しているか

### 6. 体系としての接続
- [ ] 章末が次章へ、本文中の motif（銀の弾丸＝1章／閉じた・開いた問い＝7章 など）が他章へつながっているか
- [ ] 各部の index が、その部の役割と章のつながりを示しているか

### 章別・特に注意して見る点（初稿レビューで残った弱点）
- **第1章**：第五世代→銀の弾丸の motif が⑤の誤解（小さな銀の弾丸探し）まで通っているか
- **第2章**：崩し D（三通り命名）が prose の質感を壊していないか／カーニハン不在の埋め合わせ
- **第3章**：リファクタリング≠作り直しの線引きが鈍っていないか
- **第4章**：Apollo（成功例）が章を支配していないか／③失敗の厚み
- **第5章**：①属人化の「残された側」と「握る当人」の両面が出ているか
- **第6章**：**静かな回収（A-6章 B欄）が前面化しすぎ／弱すぎないか ＝ 本全体の要**
- **第7章**：相対主義リスク（「全部アリ」への薄まり）を誤解の解体で潰せているか
