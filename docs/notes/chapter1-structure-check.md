# 第1章 構造チェック（執筆後の自己点検）

検証目的：(1)「自由」を本文で抑え着地文だけで出す技法が効くか (2) 第4章（崩しE）と並べて、不自由の地図が別の崩しでも通奏低音として効くか。

**拍の対応**

| 拍 | 本文の位置 |
|----|-----------|
| 事件 | 「この機能も入れましょう」→先輩の渋い顔 |
| ① 問題 | 機能はかけ算で絡む／誰も全体を見渡せない＝複雑さに飲まれる |
| ② 最初の解決策 | 力で勝つ。第五世代コンピュータの夢 |
| ③ 失敗 | 夢は実らず。ブルックス「銀の弾丸はない」＝複雑さは本質的 |
| ④ 新しい考え | 倒さず分ける。マキルロイ／パイプ＋出どころばらばらのKISS/DRY/YAGNI |
| ⑤ 今の当たり前 | 足す前に止まる。誤解の解体（YAGNI・DRY）＝誤解は小さな銀の弾丸探し |
| ⑥ まだ終わらない | モノリス vs マイクロサービス（開いた問い） |
| 着地＋次章橋 | 「シンプルさとは…自由だ」→ 読めなさへ（第2章へ） |

**所見**

- **「自由」抑制は成立した。** 本文では「自由」という語を一度も使わず、「飲み込まれない」「手を入れ続けられる」「手なずける」で進め、最後の着地文一文でだけ「自由」と落とした。不自由（問題側）は①と次章橋で名指したが、自由（勝ち取り側）は温存できた。狙いどおり、最後の一文が安っぽくならない。
- **崩しはC（反例常設）。** 第五世代と銀の弾丸という失敗を前景化し、「人は複雑さを一撃で消したがり、そのたび失敗する」という小さな motif を⑤の誤解（＝小さな銀の弾丸探し）まで通した。
- **第4章との対照テスト＝合格。** 第4章は「触れないコードへの恐怖」を低く鳴らすE（通奏低音）。第1章は「一撃で消したい誘惑とその敗北」を反例で見せるC。**質感はまるで違うのに、どちらも『不自由→自由』のエンジンで前へ進む。** 地図は崩しの種類を選ばず通奏低音として効く、と確認できた。
- **第2章への橋＝改題した第1部の接続を実演。** 「複雑さ」を分けても「読めなさ」が残る、で第2章（理解）へ自然に渡せた。第1部「複雑さと読めなさに抗う」が一本につながる。
- **要・出版前ファクト確認**：第五世代コンピュータの目標と帰結／ブルックス「銀の弾丸はない」と本質的・偶有的複雑さ／マキルロイとパイプ・UNIX哲学／KISS・DRY・YAGNIの出どころ。本文は事実ベースだが、刊行精度では一次資料を一度通す。
