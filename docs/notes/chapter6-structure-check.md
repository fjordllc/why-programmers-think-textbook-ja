# 第6章 構造チェック（執筆後の自己点検）── 心臓

検証目的：(1) 崩し A（歴史的場面から開く）が本書の「読者の小さな事件から開く」型を破っても機能するか (2) 本書のタイトル「自由」を**文字どおり**回収しつつ、前面化しすぎない「静かな回収」ができるか。

**拍の対応**

| 拍 | 本文の位置 |
|----|-----------|
| 事件（崩し A） | プリンタの紙詰まり。中身が非公開で、数行直すことすら許されない（ストールマンの実話を匿名化して場面化） |
| ① 問題 | 中を見られないと使う人は作り手に従属。直す力・学ぶ機会・配る権利が手元にない。「昔話ではない」と現在へ接続 |
| ② 最初の解決策 | ソフトの中身は秘密にして売る。隠すことが価値を守るという当たり前 |
| ③ 失敗 | その当たり前が使う人を無力化。バグも直せず、会社の都合で道具が死ぬ |
| ④ 新しい考え | ストールマン。四つの権利（使う・読む・直す・配る）／GNU 宣言／free＝無料でなく自由（"ただ酒でなく言論"）|
| 広がり | トーバルズ（基盤）＋変更を束ねる道具（Git を匿名化）＋誰でも書ける百科事典（Wikipedia）。英雄一人にしない |
| ⑤ 今の当たり前 | 世界の根幹が公開ソフトの上で動く。誤解二つ（タダ働きではない／フリー＝無料ではない）を内側で解く |
| ⑥ まだ終わらない | 自由をどう支え続けるか（メンテナの疲弊・ただ乗り・資金）＝開いた問い |
| 着地＋次章橋 | 「オープンソースとは直し・見て・配る自由そのものだ」→「自由な世界で、なぜ議論は終わらないのか」→ 第7章（正解はない）へ |

**所見（最重要：静かな回収の検証）**

- **タイトルの文字どおりの回収を、本の真ん中に置けた。** 第1〜5章では「自由」は着地文だけの比喩だった。本章で初めて、歴史の当事者自身が "freedom" を旗印に掲げたという事実として「自由」が出る。free＝無料でなく自由、の説明がその核。
- **前面化を抑えた工夫。** 「これが本書のタイトルだ」とは一度も書かない。free の説明のあとに「この章で扱っているのは、ずっと、その一語だ」とだけ置き、気づく読者だけが気づく静かな回収にした。叫ばないことで、かえって効く。要・編集判断：この一文が説明的すぎないか、もっと削れるか。
- **崩し A は成立した。** 唯一、読者の事件でなく歴史的場面（プリンタ）から開いた。だが「中身を開けられない機械・勝手に直せない製品」と現在へ橋を架け、読者が立てる視点を回収した。固有名（ストールマン／Xerox）は本文で出すか要判断——今回は人物名は出し、製品名は伏せた。
- **英雄分散。** ストールマン（宣言）→トーバルズ（基盤・道具）→百科事典、と複数に散らし、一人の伝記にしない。トーバルズ／Git／Wikipedia は匿名化気味に流し、章の主役を「自由という考え」に保った。
- **「自由」抑制ルールとの整合。** 他章は本文で自由を言わない方針だが、本章だけは例外的に本文で「自由」を出してよい（ここが文字どおりだから）。それでも乱発せず、free の説明と着地に絞った。
