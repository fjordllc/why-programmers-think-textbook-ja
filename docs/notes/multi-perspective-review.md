# 多視点レビュー記録 ──『プログラマーはなぜそう考えるのか』

各原稿を、次の**4つの視点＋1つの通奏基準**で通読レビューし、所見→修正→再レビューを記録する。

## レビューの視点

1. **技術書編集者** ── 構成・密度・文体・体裁。冗長や言い換え水増し、見出し作法、和欧スペース、AIっぽい言い回し、章の payoff。
2. **FBC メンター** ── 初学者への教育的正しさ。「なぜ」を渡せているか。ドグマ（「絶対ダメ」）に倒れていないか。用語の導入がやさしいか。学習者に対してフェアで励ましがあるか。
3. **エンジニア歴 30 年のベテラン** ── 事実・歴史の正確さとニュアンス。単純化しすぎていないか。断定は擁護可能か。人物・技術への敬意と公平。
4. **FBC 受講生（初学者の読者）** ── わかるか。どこで迷子になるか。未説明のジャーゴン。続きを読みたくなるか。

**通奏基準：読み物として面白いか。** ── 上の4点すべてに通底させる。正しくても退屈なら不合格。情景・対話・対比・リズム・「次が読みたい」引き。

各章 ◎=良い / △=要改善（修正した） / ▲=著者判断に委ねる（修正せず提案）で記す。
