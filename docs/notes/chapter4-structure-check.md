# 第4章 構造チェック（執筆後の自己点検 v2）

**前回からの最大の変更：崩しAを撤回した。**

前回はこの章に崩しA（入りの反転＝Apolloから冷たく開く）を割り当て、「開幕シーンが象徴場面を兼ねて経済的」と評価した。だが今回のレビューが正しく突いたのは、まさにそこだ。Apolloから開くと、読者は「Apolloの本」として記憶し、Apollo（成功例）が章を支配してしまう。

この章の本当のエネルギーは、歴史的場面ではなく、**読者がいま感じている恐怖**（＝触れないコード）にある。だから開幕は現在・読者の席に置くのが正しい。

**framework への教訓：崩しは事前割り当てではなく、章の主人公（エネルギーの在り処）から導く。** 崩しAは、エネルギーが歴史的場面そのものにある章（第6章＝GNU宣言、第7章＝goto論争など）にこそ向く。

**発見：第5の変奏「通奏低音（E）」。** この章を貫いたのは A〜D のような構造上の崩しではなく、「触れないコードへの恐怖」という一つの主題を最初から最後まで低音で鳴らし続ける手法だった。これは構造ではなく主題のレベルで効く。各章がそれぞれの「恐怖→自由」の弧を持てるなら、これが本の本当のエンジンになりうる（下記）。

**取り込んだレビューと対応**

| 指摘 | 対応 |
|------|------|
| Apolloが強すぎる／2〜3割短く（編集者） | 冷たい開幕から外し、中盤の一答に縮小 |
| 違和感をもっと早く（編集者）／受講生あるあるを前に（メンター） | 冒頭を「触らないほうがいい」と言われる現在の場面に |
| 半世紀後の転換に余韻を（編集長） | 1202 と FAIL の映像的な韻で接続 |
| Apolloは成功例すぎ、普通の失敗を（30年目） | ソフトウェア危機＋ロケットの失敗を③に追加 |
| 人物紹介が要らない、一文で（5年目） | Hamilton／Beck／Dijkstra を各一文に |
| Apolloで終わらせず名文で終わる（全員一致） | 末尾を「明日も触れる自由」に着地 |
| 主人公が入れ替わる→主人公は「触れないコード」（編集長） | 章全体をこの恐怖で貫いた＝今回の背骨 |

**副次効果：前回「窮屈」と報告した③失敗が、今回は厚くなった。** 主人公を恐怖に据えたことで、ソフトウェア危機とロケットの失敗が自然に③へ収まり、Apollo（成功）と対になった。
