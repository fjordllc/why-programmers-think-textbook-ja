# 第9章 構造チェック（執筆後の自己点検）

**拍の対応**

| 拍 | 本文の位置 |
|----|-----------|
| 事件 | 「URL ちょうだい」。何もインストールせず、どの機械でも開いた瞬間に動く当たり前のすごさ |
| ① 問題 | 機械の種類ごとに作り直し、配る道は持ち主が握る＝OS／ベンダーに縛られ届かない |
| ② 最初の解決策 | 一つの完璧な共通規格で統一する（CORBA を匿名で。分厚く厳密な標準） |
| ③ 失敗（崩し C・前景化） | 重すぎて沈んだ。仕様は分厚く、道具は高価で、参加の壁になった。「完璧を目指して重くなりすぎたもの」を章の影の主役に据える |
| ④ 新しい考え | 正反対＝ゆるく・軽く・つなぐ。HTML/HTTP/ブラウザ、住所に取りに行く軽い作法（REST）、公開仕様＝誰でも参加。後から JS・土台（Rails 的）が乗る |
| ⑤ 今の当たり前 | URL 一つで世界に届く。誤解「Web は全部 SPA にすべき」を内側で解く（軽さを自ら手放す＝重い標準の失敗の再来） |
| ⑥ まだ終わらない | Web はどこまで重くなっていいか（豊かさか逆戻りか）＝開いた問い |
| 着地＋終章橋 | 「Web とは許可なく作って届けられる自由」→ 九つの自由を列挙して束ね→「新しい書き手＝AI」→ 終章へ |

**所見**

- **崩し C（反例常設）を全篇に通した。** CORBA を「重くなりすぎて沈んだ影の主役」として③に据え、④以降も「ゆるく軽い Web」との対比で繰り返し参照、⑤誤解（全 SPA 化）まで「重い標準の失敗の再来」として一本の motif にした。第1章 C と同型だが非隣接（隣接は第8章 B）。
- **三つの motif を終章前に合流させた。** 「銀の弾丸探し（1章）」「閉じた/開いた問い（7章）」を⑤で明示的に呼び戻し、本全体を体系として閉じる準備をした（STYLE_GUIDE 0.1 の「本文中で他章へ体系として接続」）。
- **章末で本全体を束ねた。** 最終章なので、九つの自由を一息に列挙し、「最初からあったものではない／勝ち取ってきた」と総括。そのうえで「新しい書き手＝AI」を一文だけ予告し、終章へ橋を架けた。列挙が定型の「まとめ」にならないよう、見出しなしの結びとして処理（## まとめ は作らない方針を維持）。
- **固有名の抑制。** CORBA・REST・CERN・Tim Berners-Lee・JavaScript・Rails を本文では出さず機能で描いた。CERN/W3C 等は WRITING_PROMPT で「初出で軽く説明」とあるため、本文に出す場合は要・軽い説明。出すか巻末注かは要編集判断。
- **「自由」抑制、九章すべてで貫徹。** 本文は「届かない／届く」で進め、着地文だけ「自由」。最後の列挙で初めて九つの「自由」を並べたのは、終章への意図的な解放。

**要・出版前ファクト確認**
- CORBA（OMG・1991〜）の重厚さと普及の失速、WS-*／SOAP の重さと REST への移行
- REST（Roy Fielding・2000 の学位論文）の「リソースに URI を与えて取得」という核
- Web の起源（Tim Berners-Lee・CERN・1989〜1991、HTML/HTTP/URL/ブラウザ）
- JavaScript（1995）・Rails（2004）を「後から乗った層」とする時系列
- SPA を唯一解とする風潮への批判の妥当性（本書の主張として）
