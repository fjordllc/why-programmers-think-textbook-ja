# 第7章 構造チェック（執筆後の自己点検）

検証目的：(1) 第4部の枠組み「閉じた問い／開いた問い」を、説教でなく読者の体験から立てられるか (2) 相対主義リスク（「全部アリ」への薄まり）を誤解の解体で潰せるか (3) 隣接（第6章 A）と崩しが重ならないか。

**崩しの判断（OUTLINE からの変更）**

OUTLINE は第7章に崩し A（goto 論争から冷たく開く）を仮置きしていた。だが **隣接する第6章が確定的に A**（プリンタ事件から開く）なので、隣接回避ルールに従い変更。第7章は読者の事件（タブ／スペース論争）から開き、崩しは **D（実技＝有名な論争を「閉じた／開いた」に仕分けさせる）** にした。理由：この章の仕事は「閉じた／開いた」という枠組みそのものを読者に渡すことであり、抽象的な枠組みは仕分けの実技で初めて自分のものになる。goto 論争は冒頭でなく④の本文で「閉じた問いの例」として使った。

**拍の対応**

| 拍 | 本文の位置 |
|----|-----------|
| 事件 | チャットがタブ/スペースで炎上。なぜ些細なことで何年も揉めるのか |
| ① 問題 | 「正しい一つ」を求める癖。開いた問いを閉じられると思い込むと永遠に争う＝唯一解に縛られる |
| ② 最初の解決策 | 権威/勝者が「これが正解」と宣言して決着させる |
| ③ 失敗 | 「唯一の正解」は何度も覆る。第1章「銀の弾丸探し」の別の顔（motif の再登場） |
| ④ 新しい考え | 問いを二種類に分ける。閉じた問い（goto 論争＝勝者あり）／開いた問い（静的・動的型、タブ/スペース＝勝敗なし） |
| 崩し D | 世の論争を自分で「閉じた/開いた」に仕分けさせる |
| ⑤ 今の当たり前 | 議論前に種類を見分ける。最重要の誤解「正解はない＝どっちでもいい」ではないを内側で解く（閉じた問いには答えがある／開いた問いも根拠で選ぶ） |
| ⑥ まだ終わらない | どの問いがいつ閉じるかは不明。閉じた問いが再び開くことも。仕分けに最終版はない＝メタに開いた問い |
| 着地＋次章橋 | 「正解はないとは選ぶ自由を手放さない積極的な答え」→「最たる開いた問い＝どの言語で書くか」→ 第8章へ |

**所見**

- **相対主義リスクを正面から潰した。** OUTLINE が警告した「全部アリ」への薄まりを、⑤誤解で「『全部アリ』は考えるのをやめた人の言葉」と明言し、閉じた問い＝答えあり／開いた問い＝根拠で選ぶ、と二段で解いた。第7章だけ着地の動詞が「獲得」でなく「保持（手放さない）」になる、という OUTLINE の設計をそのまま実装。
- **motif の連結。** ③失敗を第1章「銀の弾丸探し」と明示的につないだ。本全体が体系として接続する（STYLE_GUIDE 0.1）狙いの実演。
- **goto／型／集中分散の配置。** goto は閉じた問いの代表として本文へ。型は開いた問いの代表。集中 vs 分散（実質 Git）は紙幅の都合で本章では落とし、Git は第6章で既出のため重複回避。要・編集判断：集中/分散を閉じた問いの二例目として足すか。
- **第6章→第8章の橋。** 第6章末「自由な世界でなぜ議論が終わらないのか」を冒頭で受け、章末で「最たる開いた問い＝言語選び」→第8章（言語をつくる）へ。
- **崩し D 三度目だが非隣接・別題材。** 命名（2章）・レビュー文面（5章）・問いの仕分け（7章）で題材が異なる。隣接は第6章 A・第8章 B（予定）なので D の重複なし。

**要・出版前ファクト確認**
- ダイクストラ「Go To Statement Considered Harmful」（1968・CACM、表題は編集の Wirth による）と構造化プログラミング
- 静的型／動的型論争が「開いた問い」であることの妥当性（本書の主張として）
- 集中型／分散型バージョン管理を「実質 Git が勝った閉じた問い」と扱う妥当性（本章では未使用・要判断）
