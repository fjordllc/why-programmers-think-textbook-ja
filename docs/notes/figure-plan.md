# 図版計画 ──『プログラマーはなぜそう考えるのか』

本書を**豊富な図版で読みやすく**するための計画。FBC_Press `FIGURE_GUIDE.md`／`ASSET_RULES.md` に準拠する。

## 方針（FBC ルール準拠）

- **自作 SVG を優先**（第三者画像のライセンス管理を避け、本書の MIT と衝突させない）。
- ファイル名 `fig-<章>-<図番号>.svg`（例 `fig-1-1.svg`／はじめに＝`fig-0-x`／終章＝`fig-e-x`）。配置先 `manuscript/assets/`（`book.toml` の追従と相対パスは生成時に確定）。
- **1 図 1 主張**。複数論点を 1 枚に詰めない。
- 図中の用語・コード断片は**本文と一致**させる。
- **色だけで意味を分けない**（ラベル・配置・形でも区別）。
- キャプションは**短く言い切る**（本文の繰り返しにしない）。alt は要点 1 文（意味を書く／キャプションの丸写しにしない）。
- 図は本文の代わりではなく補強。前後の本文が図の読み方を支える。

凡例：★＝各章の旗艦図（最優先）。種類＝流れ／対比／段階／循環／構造／空間。

## 実装メモ

- `manuscript/assets/` を作り SVG を置く。本文には `![alt](../assets/fig-x-y.svg)` ＋直後にキャプション行で挿入（mdBook の相対パスに注意。`SUMMARY.md` 階層から各章への相対で確認）。
- SVG は文字を `<text>` で持ち、フォントは CSS（theme）に委ねる。図中文字も和欧スペースを守る。
- 色は最小限（前景/背景＋1 アクセント）。意味は形・ラベル・配置で担保（色覚多様性配慮）。
- 生成手段：自作 SVG（手書き or スクリプト）。再生成できるよう、複雑な図は元の定義（このファイルの記述）を残す。
