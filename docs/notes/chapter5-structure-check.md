# 第5章 構造チェック（執筆後の自己点検）

**拍の対応**

| 拍 | 本文の位置 |
|----|-----------|
| 事件 | 「レビューに出して」と言われ、動くのに見せるのをためらう |
| ① 問題 | 一人が握ると、その人が消えたら誰も触れない＝属人化。握る当人も休めない |
| ② 最初の解決策 | エースに任せる。属人化を「頼れる強み」と見なす |
| ③ 失敗 | その一人が抜けると崩壊。引き継ぎ書は「なぜ」を書ききれない。強みが最ももろい一点だった |
| ④ 新しい考え | 書く最中から見せ合う（ペア・レビュー）／象徴：プルリクエストが見せ合いを日常の文化にした |
| 崩し D | 同じ指摘を二通り（「間違ってます」／「こうするとどうでしょう?」）、次も見せたくなるのはどちらか |
| ⑤ 今の当たり前 | レビューは粗探しの品評会ではない。誤解「全部指摘する」を内側で解く（目的は複数人が触れる状態にすること） |
| ⑥ まだ終わらない | どこまで／誰が承認／機械（自動チェック・AI 下読み）にどこまで任せるか＝開いた問い |
| 着地＋次章橋 | 「レビューとは一人の記憶から解き放つ自由」→「みんなを世界中に広げたら?」→ OSS（第6章へ） |

**所見**

- **崩し D を継承したが実現は別物。** 第2章 D は「命名を選ばせる」、本章 D は「レビュー文面を選ばせる」。題材が違うので反復感はない。隣接は第4章 E・第6章 A なので D の重複なし。D を⑤誤解（攻撃でなく共有）の直前に置き、実技がそのまま誤解の解体につながる構成にした。
- **① 問題＝属人化を両面で書いた。** 「残された側が触れない」だけでなく「握る当人も休めない／逃げられない」を加え、自由（一人に縛られない・休める）の二面性を出した。これは着地文の説得力に直結する。
- **③ 失敗の質。** 歴史的事故でなく「エースが抜ける」という、どのチームにも起こる失敗を据えた（第4章レビューの「Apollo は成功例すぎ、普通の失敗を」という教訓を踏襲）。引き継ぎ書が「なぜ」を書けない点まで踏み込んだ。
- **第4章→第6章の橋渡しを担う中継章。** 第4章末「大勢でどう信頼し合うか」を冒頭で受け、章末で「チームの共有→世界への共有」へ。第6章（OSS・心臓）へ自然に渡した。
- **「自由」抑制、四章連続で維持。**
